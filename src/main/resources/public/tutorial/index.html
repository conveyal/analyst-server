Transport Analyst

Concept

Transport Analyst enables the measurement of access to spatially distributed variables (people, jobs, schools, etc.) via public transport, walking, biking or driving. Common applications include measuring access to jobs, workforce, or key locations as an indicator of transport effectiveness. Indicators can be combined to create composite regional measures, such as a population-weighted average job access for a neighborhood or an entire city.

The application does not prescribe specific research questions. It accepts data in standard Shapefile (spatial) and GTFS (public transport) formats. Users may import and analyze any data set matching these formats.

Before continuing this tutorial be sure to login or create an account using the links above. You can jump directly to sections of the application using the links embedded in this tutorial. The demo data for Mexico City, included by default in your account, is read-only. To experiment with creating new data sets, create new a project.

Data management

Shapefiles

Shapefiles provide the basis for all measurements within Transport Analyst. Upload spatial variables either as a point or thematic data set. The numeric attributes within the data set can be summed as part of the accessibility measure (e.g. total job access from location x).

Use the Shapefile data manager, found under the "Data" menu, to upload new Shapefiles and enter easily read names and descriptions of the data set and attributes. You can also select a color for different attributes and visualize as a map overlay.

Before uploading a Shapefile be sure the file is in a common projection and datum (e.g. WGS84 or UTM) and zip the entire collection of files, including the .prj projection metadata, into a single zip archive. Upload only the zipped collection of files.


Transport Data

Public transport routes and schedules are imported from GTFS data sets using the Transport Data interface under the "Data" menu. Simply upload a valid, zipped GTFS archive and Transport Analyst will automatically create a transport model for the area included in the feed. You can view the routes in the GTFS data set by clicking the "eye" button in the feed listing.

Create as many different transport scenarios as you wish. Each scenario can be analyzed separately or compared against the others to measure accessibility impact of different transport systems, either existing or hypothetical.

When uploading GTFS data you either create a new layer containing only the data uploaded or augment an existing layer.


Creating GTFS

If GTFS is not already available an approximate transit network and schedule can be derived from a GIS data set containing route alignments and, optionally, stop locations. Export the GIS layer as a Shapefile, and zip it as before. Include a small metadata file in the zipped Shapefile. This metadata defines critical information about timing and speed. For more information on the metadata files see the documentation for geom2GTFS.

Alternatively you can use GTFS Editor, a web-based visual tool for creating and maintaining transit schedules.

OpenStreetMap

Data for pedestrians, bicycles and cars is imported directly from OpenStreetMap. To make corrections or updates visit the OSM website to learn more. Note that OSM data does not contain reliable speed information. By default Transport Analyst uses free-flow travel speeds for North American roadway classifications. To accurately model car travel speeds, data must be incorporated from a third-party source (e.g. roadway loop detectors or a regional traffic model). This data does not exist in a standard format and at present must be manually imported.

Accessibility Analysis

Single-point Analysis

The basis for all analysis in Transport Analyst is the access to or from a given point location (or origin) as a measure or what can be reached in a given time window. Single-origin analysis can be conducted by selecting the variable to measure (e.g. jobs) and clicking on the map to define a start location. A travel time and access plot is created showing the distribution of travel times to the selected location and variable.

Settings allow selection of the transport scenario, travel mode, date, time window, and variable to measure accessibility to. Travel times are reported as best case, worst case or average for the given time window. This data can be exported as a Shapefile or a tabular data set for use in external analysis applications.


Regional Analysis

Regional Analysis functions allow the automation of many single-point queries, one from each potential origin location (for example, every Census block). Once computed, regional analysis outputs show variation in travel accessibility for potential users at any location within the study area. Due to the number of queries involved these may take from minutes to hours to compute.


Aggregation and Bertaud Measure

Once analysis has been conducted at a regional scale it often makes sense to aggregate data to a larger spatial unit. For example, job access at a block-level might be aggregated to a neighborhood or even city scale.

For example, one useful and easily calculated measure is defined in Alain Bertaud's paper Cities as Labor Markets. This measures the average number of jobs that can be accessed within 60 minutes weighted by population. This enables fine-grain calculation of job access at a block level to be summed and converted to an average metric of access at a municipal scale.

To create the Bertaud measure, first calculate regional job access at the smallest spatial resolution possible (e.g. block-level job access). Next upload a Shapefile containing population figures, if not already included in the job data set, and a Shapefile with the larger geographic area you wish to aggregated up to. Then select the larger area as what you are "aggregating to" and the shapefile and attribute name containing the population figures. Then click calculate. Depending on the size of the study area and the number of points included, this may take a few minutes. The results will then be displayed on the map.
